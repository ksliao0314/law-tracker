# 法規更新查核 (law-tracker)

追蹤指定的台灣法規／命令，自「上次查核基準日」到「本次查核基準日」之間**是否經過修正**，並可產生**新舊條文對照表**（紅字底線＝新增、紅字刪除線＝刪除），支援匯出 Word。可依當事人／案件分成多個「任務」，各自有獨立的法規清單與查核排程。

- **資料以官方為準**：版本與條文皆取自[全國法規資料庫開放資料](https://law.moj.gov.tw/)（CC0 政府開放資料）。工具只負責偵測、定位、排版與連回官方原文，不自行竄改條文。
- **零相依套件**：純 Node.js 標準庫，**不需要 `npm install`**。`git clone` 後即可執行。
- **資料留在本機**：任務、清單、查核紀錄都存在本機 `data/` 目錄，不經第三方。

---

## 功能特色

- **多任務管理**：每位當事人／案件一個「任務」，各有獨立的法規清單與查核排程。
- **日期驅動、自動查核**：設好「下次查核基準日」與頻率，到期（基準日＋3 天）當天台灣早上 9 點後自動執行，免人工盯。錯過的期間下次開啟會自動補做。
- **法律＋命令全支援**：版本與條文取自全國法規資料庫開放資料。
- **新舊條文對照**：有異動的法規可逐字對照（紅字底線＝新增、紅字刪除線＝刪除），附立法理由／修正說明，一鍵連回官方原文，並可**匯出 Word**。
- **「有異動」看過自動消**：任務列表標示「有異動」，點開對照看過後列表不再持續提醒（任務內仍保留標示）。
- **試算不影響基準**：未到期時可隨時點「執行查核」看當下結果——標示「試算」、僅供參考、不更新基準日，但會記入查核歷史。
- **手動查核項目**：附表、公告等不在資料庫者，可自行記錄「前次修正日期」（純存檔、不自動比對）。
- **查核歷史**：每期結果保留可回看。
- **容錯可重試**：抓取失敗會顯示並提供重試；沿用舊資料會標示；右上角顯示資料同步狀態。
- **停用／刪除任務**：停用會保留紀錄並暫停自動查核。

## 系統需求

- **Node.js 18 以上**（建議 20 / 22 / 24）。
- macOS 提供雙擊啟動器與常駐安裝腳本；其他作業系統可直接 `node server.mjs`。

## 快速開始

```bash
git clone <本專案網址> law-tracker
cd law-tracker
node server.mjs
```
瀏覽器開 <http://127.0.0.1:7843> 即可使用。

> 首次「執行查核」時會自動下載最新的法規資料庫（約 30MB，數秒），之後同一天不重複下載。

依使用情境，選下面其中一種模式 👇

### 模式 A — 本機使用（個人電腦，會開關機／休眠）

**macOS**：直接**雙擊 `start.command`**，會啟動伺服器並自動開啟瀏覽器；關閉該視窗即停止。

- 查核時機：開頁時自動補做到期查核；程式開著時，內建排程器每天台灣早上 9 點過後也會自動查核當天到期任務。
- 電腦休眠／關機期間不查核，但**下次開啟（或下一次每日排程）會自動補做**錯過的期間（不會漏）。

### 模式 B — 伺服器常駐（例如辦公室一台不關機的 Mac mini，24 小時自動查核）

讓伺服器在背景常駐、開機自動啟動、當掉自動重啟，**不必有人開網頁就會自己查核**；同事可從區網其他電腦連線檢視。

```bash
# 1) 開放區網（依需要）
export HOST=0.0.0.0           # 開放區網（同事可連）
# 排程器預設每天台灣早上 9 點過後自動查核當天到期任務，通常不必改（可用 CHECK_HOUR 調整）

# 2) 安裝為 launchd 背景服務（macOS）
./service/install-service.command
```
安裝後：
- 伺服器登入即自動啟動、背景常駐、當掉自動重啟，免開 Terminal。
- 同事用 `http://<這台電腦的區網IP>:7843` 連線檢視（查 IP：`ipconfig getifaddr en0`）。
- 移除服務：`./service/uninstall-service.command`。

**常駐主機（Mac mini）建議設定：**
- **開啟「自動登入」**（系統設定 → 使用者與群組 → 自動登入）。背景服務是登入後啟動的（launchd LaunchAgent），若主機無人值守重開機後停在登入畫面，服務不會啟動、查核會中斷。
- **時區設為台灣**（系統設定 → 一般 → 日期與時間）。每天 9:00 的自動查核以主機本地時間為準。
- **Node 用 Homebrew 或官方安裝器安裝**（路徑如 `/opt/homebrew/bin/node`、`/usr/local/bin/node`）；常駐服務不建議用 nvm／asdf 等版本管理器的 Node（升級或移除該版本後服務可能失效）。
- 關閉系統休眠（或設為永不睡眠），確保 24 小時都在。

> ⚠️ **安全提醒**：本工具**預設沒有登入密碼**。模式 B 開放區網（`HOST=0.0.0.0`）等於「同一區網上任何裝置都能開啟並修改」資料。請僅在**信任的內部網路**使用，且**不要曝露到網際網路**（需要遠端請走 VPN／SSH 通道）。若需登入保護，可再加一組共用密碼（HTTP Basic）。

## 操作說明（使用導覽）

1. **新增任務**：左側按「＋ 新增任務」，填「任務名稱」、「前次查核基準日」（上次給客戶報告的日期，可留空）、「下次查核基準日」、「查核頻率（每 N 個月，0 = 不定期）」，按「建立並加入法規」。
2. **加入法規**：在任務頁按「加入法規」，輸入法規名稱或關鍵字搜尋後加入；若是附表、公告等查無者，選「以『…』新增為手動查核項目」。
3. **第一次查核**（新任務）分兩種：
   - 有填「前次查核基準日」→ 首次執行會**補上「前次查核結果」**並與本次比對。
   - 沒填 → 首次執行記為**「首次納入」**，抓下現行版本當基準。
4. **自動查核**：到了「下次自動查核日」（基準日＋3 天）當天台灣早上 9 點後，只要有開著系統（模式 A）或伺服器常駐（模式 B），就會自動執行該期查核並推進基準日；漏掉的期間下次開啟自動補做。
5. **手動試算**：還沒到期時，可隨時按「執行查核」看當下結果——會標「**試算**」、僅供參考、**不更新基準日**，但會記入查核歷史。
6. **看結果**：每部法規顯示「前次查核結果 / 本次查核結果 / 狀態」（有異動・無異動・首次納入・尚未查核・應手動查詢）。
7. **新舊對照**：狀態為「有異動」者按「新舊對照」，看紅字增刪與立法理由，可按「⤓ 匯出 Word 對照表」、點連結回官方原文。看過後任務列表的「有異動」提醒會自動消（任務內仍標示）。
8. **手動查核項目**：自行到官網查證後，可直接編輯該項的「前次修正日期」存檔（純紀錄、不做比對）。
9. **停用／刪除任務**：在「編輯任務」內，可「停用此任務」（保留紀錄、暫停自動查核）或「刪除此任務」。
10. **資料同步狀態**：右上角顯示「✓ 資料已同步 / ⚠ 資料同步失敗（點此重試）/ 資料基準 〔日期〕（點此更新）」。

## 設定（環境變數）

| 變數 | 預設 | 說明 |
|---|---|---|
| `PORT` | `7843` | 服務埠 |
| `HOST` | `127.0.0.1` | 綁定位址。本機用預設；常駐開放區網設 `0.0.0.0` |
| `CHECK_HOUR` | `9` | 每天自動查核的時點（本機時間整點；台灣設 `9`＝每天早上 9 點剛過就查當天到期任務，與「到期門檻」同一時間）。排程器固定每天一次、外加每次啟動補做。設 0–23 以外即關閉排程（僅開頁時查核） |

## 隱私與安全

- `data/groups.json`（你的任務清單，**含當事人名稱**）已列入 `.gitignore`，**不會被上傳到 GitHub**。其餘大型法規索引也排除（首次查核自動重建）。
- 全部資料留在本機；唯一的對外連線是向**官方全國法規資料庫／立法院開放資料**抓取公開法規。
- 部署到伺服器時務必詳閱上方「安全提醒」。

## 運作方式（簡述）

1. 每個「任務」設定「下次查核基準日」與頻率；系統在「基準日＋落差天數（預設 3 天）」之上午 9:00 後自動執行該期查核（容許法規上網時間差）。
2. 查核時逐一比對清單法規「上次正式查核所記錄的版本」與「本次基準日的在效版本」，標出有異動者。
3. 有異動的法規可開「新舊對照」：以官方批次條文（新版）對上官方歷史法規（舊版）做逐字差異，並連回官方原文。
4. 不在批次資料庫中的法規（如附表、公告）歸入「需手動查核」，由使用者自行填記前次修正日期。
